title: Overview
description: Documentation map for implementing, running, and evaluating NetOpsBench agents.
---

NetOpsBench evaluates troubleshooting agents on generated data-center network fault scenarios. A run provisions a SONiC-VS / Containerlab topology, injects a controlled fault or healthy episode, exposes runtime evidence, calls an agent, and scores the returned `DiagnosisResult` against scenario ground truth.

These docs are organized around the agent-development workflow.

## Main path

<Steps>
  <Step>
    ### Verify the runtime

    Use [Quickstart](/docs/quickstart) to check Linux, Docker, Containerlab, credentials, generated scenarios, and one XS run.
  </Step>
  <Step>
    ### Implement the agent contract

    Read [Custom Troubleshooting Agents](/docs/build-your-agent/custom-agents) for the required `diagnose(context) -> DiagnosisResult` shape.
  </Step>
  <Step>
    ### Run benchmark cases

    Use [Running Benchmarks](/docs/run-benchmarks/run-scenario-vs-suite) for one scenario, small suites, full scale runs, and multi-scale batches.
  </Step>
  <Step>
    ### Interpret results

    Use [Benchmark Methodology](/docs/run-benchmarks/methodology) for scoring definitions and [Benchmark Results](/docs/run-benchmarks/results) as a reference snapshot.
  </Step>
</Steps>
